## 16 에러 처리 시 주의 사항
  - 상세 에러를 통해 공격이 가능
  - 에러 출력 경우
    - 개발기 소스코드 그대로 운영기 적용
    - 디버깅 용도로 에러 출력 후 미 제거
    - 보안적 사고 부족
  - 정리 
    - 와스에 상세에러는 뜨면안됨
    - 에러페이지를 따로 만들어서 에러처리

## 17 주석 사용 시 주의사항
  - 주석
    - 작성, 수정, 일정, 의견, 설명등 적음  
  - 주석 내 정보
    - 개발 정보
    - 웹 서비스 계정 정보
    - 서버 계쩡 정보
    - SQL 쿼리 정보
    - 에러 정보
 ## 18 로깅 시 주의사항
  - 로깅(Logging)
    - 로그를 기록하는 행위
